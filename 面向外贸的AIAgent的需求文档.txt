# 个人外贸AI Agent助理（本地部署版）——开发需求文档

**版本**：v1.0
**日期**：2026-04-29
**文档性质**：产品需求规格说明书


## 一、项目背景与目标

### 1.1 项目背景

外贸从业者在日常工作中面临四个核心痛点：

- **选品依赖经验**：缺乏系统性数据支撑，难以持续发现高潜力商品
- **客户开发效率低**：广撒网式触达转化率低，缺乏对目标客户的深度了解
- **邮件管理碎片化**：询盘、订单、物流等邮件散落在收件箱中，查找和追溯耗时
- **客户跟进凭记忆**：缺乏统一的沟通时间线，容易遗漏关键节点，交接困难

### 1.2 项目目标

开发一套**基于本地部署的个人AI Agent助理**，实现：

1. 所有数据（邮件、客户信息、订单记录）**100%存储于本地**，保障数据主权与隐私安全
2. 以AI驱动选品研究、客户挖掘、邮件解析和订单管理四项核心能力
3. 以“客户时间线”为核心交互，串联每位客户的完整生命周期
4. 个人单机可运行，无需云服务依赖，硬件成本可控

### 1.3 核心设计理念

本系统不是工具的简单堆砌，而是一个**懂外贸流程的智能协作者**。其定位从传统的“被动记录者”升级为“主动参谋者”——不仅能记录订单，更能主动告诉用户“该卖什么”“该联系谁”“该怎么聊”，并自动将所有沟通和订单串成清晰的时间线。


## 二、系统架构总览

### 2.1 架构层次图

```
┌─────────────────────────────────────────────────────┐
│                  用户交互层                          │
│     Web UI（本地浏览器访问）  /  对话式指令入口       │
├─────────────────────────────────────────────────────┤
│                   智能编排层                         │
│   Agent 任务调度  │  工作流引擎  │  跨模块通信        │
├─────────────────────────────────────────────────────┤
│                   四大业务模块                        │
│  选品研究引擎 │ 客户开发引擎 │ 邮件与订单中心 │ 客户时间线 │
├─────────────────────────────────────────────────────┤
│                   基础能力层                          │
│  本地 LLM 推理  │  邮件协议适配  │  网页抓取与搜索    │
├─────────────────────────────────────────────────────┤
│                   数据持久层                          │
│  SQLite/PostgreSQL  │  向量数据库  │  本地文件存储     │
└─────────────────────────────────────────────────────┘
```

### 2.2 部署模式

- **部署方式**：本地单机部署（Windows 10+/macOS/Linux均可）
- **访问方式**：通过浏览器访问本地运行的Web界面，或在手机端通过Telegram/飞书等通道发送指令，Agent在电脑上执行操作并返回结果
- **网络要求**：基础功能可在断网环境下运行；选品数据采集和邮件收发需要互联网连接
- **数据存储**：所有业务数据存储在本地数据库和文件系统中，不经过任何第三方云端

### 2.3 技术栈选型概览

| 层级 | 推荐方案 | 备选方案 |
|------|---------|---------|
| LLM推理框架 | **Ollama**（极简部署，一键运行开源模型） | llama.cpp（资源更优，适合老旧硬件） |
| 推荐基础模型 | Qwen2.5（14B/7B），中文优化、Apache 2.0商用许可 | DeepSeek（MoE架构，性能更优） |
| Agent开发框架 | **Dify**（可视化编排，低代码构建Agent工作流） | LangChain（更灵活，适合代码导向） |
| 知识库（RAG） | LlamaIndex + 本地向量数据库（Qdrant/Chroma） | Dify内置知识库 |
| 自动化工作流 | **n8n**（开源，可本地部署，300+集成节点） | 自研调度模块 |
| 后端服务 | Python（FastAPI） | Node.js（Express） |
| 前端界面 | React/Vue.js | Dify内置前端 |
| 业务数据库 | PostgreSQL | SQLite（轻量级方案） |
| 向量数据库 | Qdrant（本地部署） | Chroma（嵌入式，零配置） |
| 邮件集成 | 通过Gmail API / IMAP协议直连 | n8n内置邮件节点 |

**选型原则**：优先选择开源、本地可部署、社区活跃的方案。个人使用场景下，轻量化优于企业级能力，易维护性优于极致性能。Ollama是目前在消费级机器上本地运行LLM的门槛最低方案，Ollama+DeepSeek/Qwen+Dify组合可实现零成本搭建私有AI Agent。


## 三、硬件配置建议

| 组件 | 最低配置 | 推荐配置 |
|------|---------|---------|
| CPU | 4核8线程 | 8核16线程 |
| 内存 | 16GB DDR4 | 32GB DDR5 |
| 显卡 | NVIDIA RTX 2060（6GB显存） | NVIDIA RTX 4090（24GB显存） |
| 存储 | 256GB NVMe SSD | 1TB NVMe SSD |

**说明**：最低配置可运行7B参数的量化模型（如Qwen2.5-7B-Q4_K_M），实现基础AI能力。推荐配置可流畅运行14B以上模型，获得更好的智能分析效果。Ollama支持GGUF量化格式，70B模型仅需约8GB内存即可运行，7B模型仅需约4GB。

**若本地硬件不足的替代方案**：可先使用第三方LLM API（如通义千问、DeepSeek等国内大模型API，成本极低），整体架构不变，仅需将模型调用节点指向云端API即可。业务数据仍全部留存本地，这是与SaaS产品的本质区别。


## 四、核心功能需求详述

### 4.1 模块一：智能选品研究引擎

#### 4.1.1 功能概述
基于用户设定的品类关键词和市场偏好，定期采集全网趋势数据，结合利润测算，输出选品建议报告。

#### 4.1.2 详细需求

**需求1：全网趋势扫描**
- 按用户设定的品类关键词，定期（可配置频率：每日/每周）抓取以下数据源：
  - Google Trends 趋势数据（通过第三方API或公开数据）
  - Amazon Best Sellers 榜单（目标站点）
  - AliExpress/1688热门品类与价格区间
  - 行业社交媒体（如TikTok、Instagram）相关话题热度
- 识别需求上升的商品品类，标注趋势强度（上升/稳定/下降）

**需求2：竞品动态监控**
- 用户可配置关注竞品的官网、社媒账号
- Agent定期抓取竞品新品上架、价格变动、促销策略等动态
- 自动汇总竞品产品策略变化，生成简要分析

**需求3：利润初步测算**
- 结合1688等平台的采购参考价和目标市场零售参考价
- 自动计算毛利预估（扣除预估运费、平台佣金等）
- 提示相关认证要求（如CE、FDA、FCC等，基于目标市场）

**需求4：报告输出**
- 生成“每周选品机会报告”，包含：
  - 推荐商品及推荐理由
  - 数据来源链接
  - 利润估算参考
  - 风险提示（竞争激烈度、季节性等因素）
- 支持推送到用户邮箱或直接在对话界面查看

**需求5：定时自动运行**
- 用户可设置定时任务，Agent在后台按设定周期自动执行上述采集
- 支持手动触发“立即研究”指令

#### 4.1.3 AI能力要求
- 综合分析多种数据源，提炼研究洞察
- 判断趋势信号的可靠程度
- 生成结构化的推荐报告


### 4.2 模块二：客户开发与联系引擎

#### 4.2.1 功能概述
基于选品结果或用户指定的目标市场，在公开渠道搜索潜在采购商，结合公开信息研究客户背景，生成个性化开发信草稿。

#### 4.2.2 详细需求

**需求1：客户线索挖掘**
- 根据用户设定的客户画像条件，在LinkedIn等公开来源搜索潜在客户
- 搜索条件可包括：行业、地区、公司规模、职位关键词
- 提取公开联系信息（如企业公开邮箱、官网联系方式页面）
- **重要合规约束**：发送连接邀请、主动私信等操作需用户手动确认执行

**需求2：重点客户深度研究**
- 对用户标记的“重点目标客户”，Agent自动采集并分析：
  - 公司官网信息与产品线
  - 近期新闻动态、财报摘要（如公开）
  - 关键决策人的社媒公开动态（LinkedIn帖子、Twitter等）
- 提炼破冰话题建议，例如：“该公司刚完成B轮融资，正扩张北美产品线，建议以此为切入点”

**需求3：个性化开发信生成**
- 输入：目标客户研究摘要 + 推荐主推产品
- 输出：多个风格版本的开发信草稿（正式商务/简洁直白/故事型），不以风格为名编造事实
- 特征：
  - 每个版本基于客户背景和产品卖点个性化生成，非模板套用
  - 附带简要说明：为什么这样写、适用场景
  - 输出时可同时给出邮件主题行/标题建议
- 用户可微调后复制使用，Agent会记录该版本文本以供后续参考

**需求4：客户分类管理**
- 按开发阶段分类：潜在线索 → 已联系 → 已回复 → 询盘客户 → 成交客户
- 支持标签系统：用户可自定义标签（如“重点跟进”“价格敏感”“样品阶段”等）


### 4.3 模块三：统一沟通与订单中心

#### 4.3.1 功能概述
连接用户的业务邮箱，自动分拣外贸相关邮件，提取关键信息，生成报价草稿，自动创建订单记录。这是系统的核心枢纽，将非结构化的邮件转化为结构化业务数据。

#### 4.3.2 详细需求

**需求1：邮件集成**
- 支持连接多个邮箱账号（如Gmail、Outlook、企业邮箱等）
- 认证方式：OAuth 2.0（Gmail）或 IMAP/SMTP协议
- 所有邮件数据仅存储在本地数据库，不经过任何云端中转

**需求2：智能邮件分拣**
- 自动识别邮件类型并分类：
  - **新询盘**：首次联系的潜在客户
  - **订单确认**：包含PO、PI、订单确认信息
  - **付款通知**：水单、转账凭证
  - **货运文件**：B/L、运单号、发货通知
  - **老客户沟通**：已有订单的跟进邮件
  - **垃圾/非业务邮件**：自动归档，不在业务视图中显示
- Agent通过邮件元信息（发件人、主题关键词、是否回复过）和邮件内容综合判定分类

**需求3：关键信息提取**
- 从邮件正文和附件（PDF、图片、Word）中自动提取：
  - 产品型号/名称、数量、目标价格区间
  - 交期要求、目的地、运输方式偏好
  - 客户公司名称、联系人及职位
- 提取结果以结构化摘要展示，不替代阅读原文

**需求4：报价草稿生成**
- 在回复询盘时，Agent可调用产品参数和报价模板
- 自动填入产品名称、规格、MOQ、阶梯价格等
- 生成专业报价单（PDF格式），含公司抬头和条款占位符
- 用户确认后可一键复制或发送

**需求5：订单记录管理**
- 两种创建方式：
  - **自动创建**：Agent识别到订单确认邮件后，自动解析并创建订单记录
  - **手动创建**：用户可在订单看板手动新增订单
- 订单信息字段：客户名称、产品明细、数量、单价、总金额、币种、交期、付款方式、当前状态
- 状态流转：待确认 → 待付款 → 生产中 → 待发货 → 已发货 → 已完成
- 支持按状态、客户、时间范围筛选和搜索


### 4.4 模块四：客户时间线

#### 4.4.1 功能概述
这是系统的核心交互界面。以客户为维度，自动串联所有关联事件（邮件、社交互动、开发信、报价、订单等），形成一条完整的时间叙事链，并智能生成跟进建议。

#### 4.4.2 详细需求

**需求1：自动叙事编织**
- 对于任一客户，Agent将自动串联所有关联事件，以时间轴形式展示：
  - `[2025-01-15]` LinkedIn 初次联系（如果是通过LinkedIn开发的）
  - `[2025-01-18]` 发送开发信（记录开发信内容摘要）
  - `[2025-02-03]` 客户询盘：产品A × 5000件，目标价$3.5/pc
  - `[2025-02-05]` 我方报价：$4.2/pc，交期45天
  - `[2025-02-20]` 客户回复目标价：$3.8/pc
  - `[2025-03-10]` 打样确认
  - `[2025-04-01]` 下达PI，30%定金已收
- 事件来源自动标注（邮件/手动录入/LinkedIn等）

**需求2：跟进建议生成**
- 基于时间线中的停滞节点，Agent智能判断并提醒：
  - “客户B已收到样品3周，建议本周五进行一次跟进，可询问测试反馈。”
  - “客户C报价后15天无回复，建议发送一次价值传递内容，而非直接催单。”
- 跟进建议附带推荐的沟通要点，用户可一键生成跟进邮件草稿

**需求3：多维度视图**
- 客户列表视图：按最近活动时间、订单金额、开发阶段排序
- 全局时间线视图：按时间顺序展示所有客户的最新动态（类似“动态消息”）
- 统计面板：各阶段客户数量、本月新增询盘、待处理订单等关键指标

**需求4：数据导入导出**
- 支持导入历史邮件（EML/MSG格式或通过邮箱连接导入）
- 导入时Agent自动解析历史邮件，生成初始时间线
- 支持导出客户数据为CSV/Excel，便于存档或交接
