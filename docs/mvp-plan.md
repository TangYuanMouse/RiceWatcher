# RiceWatcher MVP 规划

版本: v0.1
日期: 2026-04-29

## 1. MVP 目标

在 4 周内交付一个可演示、可扩展、可本地运行的最小系统:

- 有统一网关入口
- 有可视化客户时间线
- 能执行基础 Agent 回路并流式返回
- 能管理询盘到订单的最小状态

## 2. MVP 范围（必须做）

### A. 网关与会话
- 单 Gateway API
- session_key 串行执行
- run_id 生命周期追踪
- SSE 流式事件输出

### B. 客户与时间线
- 客户列表
- 客户事件时间线（邮件/报价/订单/跟进）
- 事件来源标注

### C. 统一询盘与订单（最小字段）
- 询盘结构化字段提取（先人工/模拟输入）
- 订单状态: 待确认 -> 待付款 -> 生产中 -> 待发货 -> 已发货 -> 已完成
- 按客户和状态筛选

### D. Agent 执行面板
- 对话输入
- 实时事件流查看
- 工具调用轨迹可视化

### E. 本地部署能力
- 后端 FastAPI
- 前端 React + Vite
- SQLite 默认存储

## 3. 延后到 V1 的能力（不进 MVP）

- 全量真实邮件协议接入
- LinkedIn 自动化抓取
- 复杂利润测算和认证库
- 多通道移动入口
- 向量检索与长期记忆增强

## 4. 四周迭代建议

Week 1
- 网关骨架、会话队列、SSE、前端控制台雏形

Week 2
- 客户/订单模型、时间线 API、前端主界面

Week 3
- 选品与开发模块最小流程、任务调度

Week 4
- 权限审批、日志审计、部署打包、演示脚本

## 5. MVP 验收标准

- 提交一条指令后 1 秒内返回 accepted 状态
- SSE 可持续推送 lifecycle/assistant/tool 三类事件
- 同一 session_key 的并发请求不会交叉写入
- 客户时间线页面可查看至少 20 条样例事件
- 订单状态流转完整可追踪
